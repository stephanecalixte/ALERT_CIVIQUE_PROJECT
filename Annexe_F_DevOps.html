F.1 — Architecture de déploiement

Vue d'ensemble des services, ports et technologies du projet Alert-Civique
ServiceTechnologiePortRôleCommande de démarrage
API REST Backend Spring Boot 4.0.3 / Java 17 9090 Auth JWT, signalements, livestreams, RBAC mvn spring-boot:run
Serveur Node.js Média Node.js 20 / Express + Socket.IO 9091 Upload vidéo multipart, WebSocket chat, MongoDB npm run dev
Dashboard Admin React 18 / Vite 5 / TypeScript 3001 Gestion incidents, utilisateurs, livestreams npm run dev
App Mobile React Native 0.83 / Expo SDK 55 19000 Interface citoyen : alertes, photo, vidéo, SOS npx expo start
Base de données principale MySQL 8.0 / MariaDB 3306 Persistance utilisateurs, signalements, rôles — (service système)
Base de données messages MongoDB 7.x 27017 Messages chat temps réel, livestreams MongoDB — (service système)
Jenkins (CI/CD) jenkins/jenkins:lts-jdk17 8081 Pipeline CI/CD : test → build → deploy docker-compose -f docker-compose.ci.yml up
SonarQube (qualité) sonarqube:lts-community 9000 Analyse statique, couverture, dette technique docker-compose -f docker-compose.ci.yml up
Flux de démarrage local :
1. MySQL démarre (port 3306) → 2. Spring Boot démarre (port 9090) → 3. Node.js démarre (port 9091) → 4. Admin React démarre (port 3001) → 5. Expo démarre (port 19000) → 6. App mobile se connecte via Expo Go

F.2 — Docker & Conteneurisation

Dockerfiles et docker-compose pour le développement, les tests et la CI/CD

docker-compose.ci.yml

Orchestration CI/CD : Jenkins (port 8081) + SonarQube (port 9000) + PostgreSQL (base SonarQube)
Docker Compose CI/CD 
version: '3.9'

services:
  jenkins:
    image: jenkins/jenkins:lts-jdk17
    container_name: alert_civique_jenkins
    ports:
      - "8081:8080"   # Jenkins UI accessible sur http://localhost:8081
      - "50000:50000" # Port agent Jenkins (builds distribués)
    volumes:
      - jenkins_data:/var/jenkins_home  # Persistance config Jenkins
      - .:/workspace:cached             # Montage du code source
    environment:
      - JAVA_OPTS=-Dhudson.plugins.git.GitSCM.ALLOW_LOCAL_CHECKOUT=true -Djenkins.install.runSetupWizard=false
    restart: unless-stopped
    privileged: true  # Docker-in-Docker si nécessaire

  sonarqube:
    image: sonarqube:lts-community
    container_name: alert_civique_sonarqube
    ports:
      - "9000:9000"   # SonarQube dashboard
    environment:
      - SONAR_JDBC_URL=jdbc:postgresql://postgres/sonarqube
      - SONAR_JDBC_USERNAME=${SONAR_JDBC_USERNAME:-sonar}
      - SONAR_JDBC_PASSWORD=${SONAR_JDBC_PASSWORD:-sonar}
    depends_on:
      - postgres
    restart: unless-stopped

  postgres:
    image: postgres:15
    container_name: alert_civique_postgres
    environment:
      - POSTGRES_USER=${POSTGRES_USER:-sonar}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD:-sonar}
      - POSTGRES_DB=sonarqube
    volumes:
      - sonarqube_data:/var/lib/postgresql/data  # Persistance données SonarQube
    restart: unless-stopped

volumes:
  jenkins_data:     # Données Jenkins (jobs, plugins, historique)
  sonarqube_data:   # Données SonarQube (projets, analyses, métriques)
Commandes CI/CD :
• Démarrer : docker-compose -f docker-compose.ci.yml up -d
• Arrêter : docker-compose -f docker-compose.ci.yml down
• Jenkins : http://localhost:8081 — SonarQube : http://localhost:9000

alert_civique_back/Dockerfile

Build multi-étages — Stage 1 : compilation Maven (Java 17) → Stage 2 : runtime OpenJDK slim
Dockerfile Spring Boot 
# ── STAGE 1 : Build ───────────────────────────────────────────────────────────
FROM maven:3.9-eclipse-temurin-17 AS builder
WORKDIR /app
COPY . .
RUN mvn clean package -DskipTests   # Compilation sans tests (exécutés en CI)

# ── STAGE 2 : Runtime ─────────────────────────────────────────────────────────
FROM openjdk:17-jdk-slim
WORKDIR /app
COPY --from=builder /app/target/alert_civique.jar app.jar  # Copie du JAR compilé
EXPOSE 8080

CMD ["java", "-jar", "app.jar"]

alert_civique_front/Dockerfile

Image Node.js 18 Alpine — Installation Expo CLI — Démarrage avec tunnel Expo (accès sans pare-feu)
Dockerfile Expo 
FROM node:18-alpine
WORKDIR /app

# Installation globale d'Expo CLI
RUN npm install -g expo-cli

# Installation des dépendances
COPY package.json package-lock.json ./
RUN npm install

COPY . .

# Ports Expo : Metro Bundler (19000), Expo DevTools (19001), Expo Web (19002)
EXPOSE 19000 19001 19002

# --tunnel : tunnel ngrok pour accès depuis appareils mobiles sans réseau local partagé
CMD ["npx", "expo", "start", "--tunnel"]

alert_civique_front/start-device.sh

Lancement Expo sur appareil Android réel — Détection ADB USB/WiFi, démarrage Node.js en arrière-plan
Bash Script Android 
#!/usr/bin/env bash
# Lance Expo sur un vrai appareil Android connecté en USB ou via WiFi

ADB="$HOME/AppData/Local/Android/Sdk/platform-tools/adb.exe"

echo ""
echo "=== Alert Civique — Démarrage sur appareil réel ==="
echo ""

# Vérifier que ADB existe
if [ ! -f "$ADB" ]; then
  echo "ERREUR : adb.exe introuvable."
  echo "Installe Android Studio : https://developer.android.com/studio"
  exit 1
fi

# Lister les appareils connectés
DEVICES=$("$ADB" devices | grep -v "List of" | grep "device$")

if [ -z "$DEVICES" ]; then
  echo "Aucun appareil détecté."
  echo ""
  echo "Options :"
  echo "  1) Branche ton téléphone en USB et accepte le débogage USB"
  echo "  2) Connexion WiFi : entre l'IP dans Paramètres → Options développeur → Débogage sans fil"
  echo ""
  read -p "IP:PORT du téléphone (laisser vide pour annuler) : " WIFI_ADDR
  if [ -n "$WIFI_ADDR" ]; then
    "$ADB" connect "$WIFI_ADDR"
    sleep 2
    DEVICES=$("$ADB" devices | grep -v "List of" | grep "device$")
  fi
fi

if [ -z "$DEVICES" ]; then
  echo ""
  echo "Toujours aucun appareil. Vérifie :"
  echo "  - Débogage USB activé (Paramètres → Options développeur)"
  echo "  - Câble USB branché + popup 'Autoriser' acceptée sur le téléphone"
  exit 1
fi

echo "Appareil(s) détecté(s) :"
echo "$DEVICES"
echo ""

# Démarrer le serveur Node.js en arrière-plan
echo "Démarrage du serveur Node.js en arrière-plan..."
cd "$(dirname "$0")/server" && node server.js &
SERVER_PID=$!
echo "Serveur Node.js PID: $SERVER_PID"
cd "$(dirname "$0")"

echo ""
echo "Lancement d'Expo..."
npx expo start --android
